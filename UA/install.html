Системні вимоги сервера
  • Процесор: не меншь ніж 1.2 ГГц / 2.0 ГГц;
  • Оперативная пам’ять: не меньш ніж 256 МБ / 1 ГБ;
  • JAVA: JRE 7+;
  • ОС: FreeBSD, Linux, MacOSX / Debian GNU/Linux 7;
Системні вимоги клієнта
  • Процесор: не меншь ніж 1.6 ГГц / 2.0 ГГц;
  • Оперативная пам’ять: не меньш ніж 512 МБ / 1 ГБ;
  • Відео пам’ять: не меньш ніж 8 МБ / 16 МБ;
  • JAVA: JRE 7+;
  • ОС: Windows 2000-8, Linux, MacOSX, FreeBSD;
  • Наявність графічної системи;
Інсталяція

Для мінімальної роботи системи створені готові комплекти сервера і клієнта системи для поточного випуску a2 (див. посилання у розділі "Загальна інформація"). Розпакуйте пакети програм до окремих тек. Тепер можна перейти до налаштувань.

Налаштування сервера

Після компіляції слід перенести виконавчі файли програми RibbonServer разом з бібліотеками в окрему теку. Також до цієї теки слід перенести файл server.properties, який постачається з проектом RibbonServer. Це головний файл конфігурації сервера без якого система не зможе працювати.

Перед запуском сервера потрібно змінити деякі налаштування у головному файлі конфігурації. Для першого тестового запуску сервера достатньо вказати шлях до теки, де сервер може створити базу повідомлень (параметр base_path). Нижче зведені усі налаштування які доступні через цей файл:

Параметр Тип даних Порогові значення Опис
Налаштування бази повідомлень
base_path Рядок -- Вказує на теку в якій зберігається база повідомлень. Цей параметр не перевіряється системою, але, при невірному шляху до теки, сервер зупинить свою роботу з помилкою. Слід створити окрему порожню теку та вказати її шлях тут.
base_allow_attachments Булеве значення 0 чи 1 Дозволяти користувачам системи закріпляти окремі файли за повідомленнями. За замовчуванням вимкнено.
[ФУНКЦІОНАЛ НЕ РЕАЛІЗОВАНО]
Налаштування мережі
networking_port Ціле Від 1 до 65535 Вказує серверу який мережевий порт потрібно слухати для отримання команд. На *nix системах щоб слухати порти від 1 до 1023 потрібно запускати сервер з правами суперкористувача root. За замовчуванням вказує на порт 3003.
networking_allow_remote Булеве значення 0 чи 1 Дозволяти з'єднуватись зовнішнім хостам. Якщо вимкнено, то сервер буде приймати з'єднання лише від адреси 127.0.0.1. Використовується для тестових цілей. За замовчуванням система дозволяє зовнішні з'єднання.
networking_max_connections Ціле -1 та від 1 до 2000 Вказує серверу максимальну кількість одночасних мережевих з'єднань. Після досягнення межі, сервер буде закривати усі нові з'єднання. Використовується для тестових цілей. Значення -1 вимикає перевірку кількості з'єднань до сервера. Значення за замовчуванням: -1.
Налаштування кешу
cache_enabled Булеве значення 0 чи 1 Вмикає механізм кешування повідомлень. За замовчуванням вимкнено.
[ФУНКЦІОНАЛ НЕ РЕАЛІЗОВАНО]
cache_size Ціле Від 1 до 1000 Розмір кешу у повідомленнях. За замовчуванням дорівнює 10.
[ФУНКЦІОНАЛ НЕ РЕАЛІЗОВАНО]
Налаштування доступу
access_all_mask Рядок Від 000 до 111.
Довжина 3 символи, дозволені тільки 0 та 1 		Визначає глобальні дозволи для спеціальної групи ALL. Ця маска доступу впливає лише на ті напрямки, в листах доступу яких не означені окрема права групи ALL. Значення за замовчуванням - 100.
Цей параметр перевіряється на коректність системою - сервер зробить попередження при запуску, якщо маска не відповідає формату.
access_allow_multiplie_login Булеве значення 0 чи 1 Визначає чи може користувач одночасно працювати на кілька з'єднаннях одночасно. Якщо цей параметр увімкнено система буде перевіряти при кожній аутентифікації чи є вже у цього користувача активні з'єднання. У разі, якщо користувач вже автентифікований, він не зможе повторно увійти у систему з іншого з'єднання. Це можна використовувати для того щоб не дозволяти використовувати один логін і пароль одночасно на багатьох комп'ютерах. За замовчуванням цей параметр вимкнено.
access_enable_sessions Булеве значення 0 чи 1 Дозволяти користувачам входити до системи за ідентефікатором попередньої сесії. Увімкнено за замовчуванням.
access_session_count_max Ціле Від 1 до 100 Кількість максимального використання ідентифікатора сесії. Після вичерпання цього ліміту, сервер не прийме ідентифікатор та змусить користувача надати свій логін і пароль. За замовчуванням дорівнюється 10.
access_allow_remote Булеве значення 0 чи 1 Дозволити використовувати видалений режим у мережевих з'єднаннях. Необхідно для роботи веб-інтерфейсу та інших фасадів системи. За замовчуванням увімкнено.
access_remote_group Рядок -- Вказує на групу користувачів якій дозволено використовувати видалений режим. За замовчуванням вказує на групу ADM.
Цей параметр перевіряється на коректність системою - сервер аварійно завершить роботу з помилкою, якщо групи не існує.
Загальні налаштування
opt_post_init Булеве значення 0 чи 1 Випускати тестове повідомлення при запуску сервера. За замовчуванням увімкнено.
opt_create_reports Булеве значення 0 чи 1 Створювати текстові рапорти щодо роботи системи і стану бази повідомлень. За замовчуванням увімкнено.
Налаштування імпорту/експорту
io_enabled Булеве значення 0 чи 1 Увімкнути імпорт та експорт повідомлень на сервері. За замовчуванням увімкнено.
io_ignore_dirty Булеве значення 0 чи 1 Ігнорувати спроби увімкнути "брудний режим" на сервері. За замовчуванням вимкнено.
ПОПЕРЕДЖЕННЯ: цей параметр не слід вмикати на робочої системі!
io_import_em_dir Рядок -- Вказує на аварійний напрямок, який буде використовуватись, якщо модуль імпорту не може визначити на який напрямок випускати повідомлення. За замовчуванням вказує на напрямок СИСТЕМА.Загублене.
Цей параметр перевіряється на коректність системою - сервер аварійно завершить роботу з помилкою, якщо напрямку не існує.
io_export_quene_size Ціле Від 1 до 200 Розмір черги повідомлень для експорту. За замовчуванням дорівнює 30.
[ФУНКЦІОНАЛ НЕ РЕАЛІЗОВАНО]
io_export_errquene_size Ціле Від 1 до 200 Розмір аварійної черги повідомлень експорт котрих було завершено помилкою. За замовчуванням дорівнює 10.
[ФУНКЦІОНАЛ НЕ РЕАЛІЗОВАНО]
Налаштування дебагінгу
debug_post_exceptions Булеве значення 0 чи 1 Увімкнути випуск повідомлень при отриманні виключення на сервері. За замовчуванням увімкнено.
debug_post_dir Рядок -- Вказує на напрямок на який будуть випускатись повідомлення про виключення. За замовчуванням вказує на напрямок СИСТЕМА.Помилки.
Цей параметр перевіряється на коректність системою - сервер аварійно завершить роботу з помилкою, якщо напрямку не існує.

Видаляти параметри з файлу конфігурації не рекомендовано!

Після початкового налаштування можна запускати сервер, який запускається командою java -jar RibbonServer.jar. Після цього сервер самостійно завантажить файл конфігурації та створить за вказаним шляхом базу повідомлень (докладніше про схему бази повідомлень див. розділ "Структура бази повідомлень"). Після цього сервер відкриває мережевий порт для клієнтських програм.

Перевірка роботи системи

Щоб перевірити роботу системи, потрібно з'єднатись до неї за допомогою клієнтської програми. Для цього можна використовувати декілька програм: RibbonProber та RibbonClient. Перша дозволяє перевірити роботу мережевого протоколу. Друга - повноціний клієнт системи.
