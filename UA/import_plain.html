Версія специфікації: 1
Тип модулю: PLAIN
Ознака модулю: IMPORT_PLAIN
Проєкт: ImportPlain

Опис модуля

Даний модуль використовується для імпорту простих текстових файлів з локальних тек чи примонтованих мережевих тек. Модуль шукає у вказаної теці файли з вказаною маскою (без рекурсивного пошуку) і імпортує їх до системи. Після вдалого імпорту, модуль вилучає файл з теки для запобігання повторного імпорту. Якщо файл неможливо вилучити, модуль додає його у чорний список і перемикає систему у "брудний" режим. Також якщо модуль не може отримати доступ за шляхом до вказаної теки, то він також перемкне систему у "брудний" режим. Випускаючи повідомлення, модуль додає свою ознаку до повідомлення з унікальним ідентифікатором.

В цьому модулю реалізовані можливості самостійного виправлення своєї роботи.

Параметри модуля

Список параметрів модулю (лише специфічні для цього модуля):
  • plain_path [Рядок] - шлях до теки звідки буде проводитись імпорт файлів;
  • plain_mask [Рядок] - маска за якою будуть шукатись файли (*.txt наприклад);
  • plain_ignore_mask [Рядок] - маска файлів які будуть автоматично ігноруватись під час експорту (some???.txt наприклад). Повинна логічно не перекривати маску пошуку (не повинна бути *);
  • plain_dir[Рядок] - вказує на напрямок випуску імпортованого повідомлення;
  • plain_copyright_override [Рядок] - вказує нові авторські права при імпорті повідомлення;
Приклад конфігурації: 
import_type=PLAIN
import_name=Test-stand
import_print=033bd94b1168d7e4f0d644c3c95e35bf
import_timeout=1
plain_path=/mount/dc-1/messages
plain_dir=Імпорт.PLAIN
plain_mask=*
plain_copyright_override=root
opt_log=1

Встановлення і запуск
  1. Завантажте цей модуль разом з усіма залежностями.
  2. Скомпілюйте файл модуля за допомогою NetBeans.
  3. Покладіть файл модулю у теку imports за шляхом де встановлено RibbonServer.
  4. Відредагуйте файл example.import за своїм бажанням та перенесіть його до теки import у базу RibbonServer. При зміні назви схеми потрібно змінити ще й назву файлу.
  5. Переконайтеся, що користувач за яким запускається сервер має дозвіл на читання та запис за шляхом імпорту.
  6. Перезавантажте сервер системи.
