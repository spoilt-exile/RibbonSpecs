Система обробки інформаційних повідомлень "Стрічка"

"Стрічка" - це новітня система обробки повідомлень для використання у інформаційних агентствах та інших виданнях. Основні функції системи полягають у структуризації повідомлень за технологічними напрямками/директоріями. Система дозволяє випускати, редагувати та видаляти повідомлення на інформаційних напрямках. Також повідомлення можливо помічати тегами. Доступ до напрямків контролюється гнучкою системою доступу. "Стрічка" цілком написана на мові програмування Java (з використанням OpenJDK 7). Код системи розповсюджується на умовах ліцензії GNU GPLv2.

Список програм з пакету системи "Стрічка":
  • RibbonServer - сервер системи;
  • RibbonProber - дебагер мережевого протоколу;
  • RibbonLastMessage - програма перегляду останнього повідомлення;
  • RibbonClient - програма-клієнт;
  • RibbonWeb - веб-інтерфейс до системи;
  • JTVProg - подготовка та випуск телепрограми до системи;
Список бібліотек:
  • libRibbonData - бібліотека структур даних;
  • libRibbonApp - бібліотека ядра програм системи "Стрічка";
  • libRibbonIO - бібліотека вводу/виводу повідомлень;
Список модулів імпорту:
  • ImportPlain - модуль імпорту текстових файлів [СПЕЦИФІКАЦІЯ];
Список модулів експорту:
  • ExportPlain - модуль експорту текстових файлів [СПЕЦИФІКАЦІЯ];
Наразі сформовано декілька тестових випусків сервера системи системи "Стрічка" - a2 bionic-rc1 (окремо позначено тегом bionic-slice-rc1 у репозиторіях). Для інших програм бажано використовувати останні версії з репозиторію.

Відомі обмеження поточної реалізації системи:
  • Загальне: протокол адміністратора не описано у специфікації;
  • Загальне: вразливий протокол (користувач може скомпрометувати систему написавши команду END: + БУДЬ_ЯКА_КОМАНДА у повідомленні);
  • Загальне: поточна реалізація libRibbonIO не дозволяэ поновити експорт повідомлення при помилці;
  • Сервер: система не може змінювати свої налаштування без перезавантаження;
  • Клієнт: можливі помилки з синхронізацією відправлення повідомлень через мережу (не перевірялось);
  • Клієнт: немає навігації за тегами;
  • Веб-інтерфейс: немає можливості переглядати напрямки;
  • Веб-інтерфейс: немає механізму сповіщень для користувача;
